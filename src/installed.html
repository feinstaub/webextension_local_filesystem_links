  1. Download and install the native application for your system.

Why do I get a warning that my computer was protected by Windows? Show answer

At the moment, the Windows installer is not signed that's why you're getting the message. Signing is planned but not ready yet. You can install the app if you're clicking on show more information and afterwards click on the new button "run anyway".

If you're concerned about security then you can do the following to be sure that everything is as usual:

  • Check that the installer is downloaded from github repo by hovering over the download links above for your OS. Repo link AWolf81/webextension_local_filesystem_links-native-host-binaries
  • Run your virus scanner on the downloaded file before executing

Why do I need to install a program? Show answer

In previous versions of Firefox (before FF 57) it was possible to access the local filesystem with the browser API. That's not supported anymore and it is similar to other browsers (e.g. Chrome has also this restriction).

To make this extension possible we need a way to interact with the file system again. This is possible with a native application that communicates with the browser with an API. It's called native messaging. The application will use the system file explorer (e.g. on Windows - it's Windows Explorer) to open the application in your linked application.

Whitelist needs to be according to new schema e.g. *://*.jsfiddle.net/*

The new format is <scheme>://<host><path> (wildcard char * is possible)
A single * will be replaced with <all_urls> by the extension. Please have a look at the following docs. for more details about match patterns

How can I whitelist a server with a port e.g. servername:8080?

Just add it with-out the port and it will be correctly handled. e.g. http://servername/*
